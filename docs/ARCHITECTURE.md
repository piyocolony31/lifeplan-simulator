# システムアーキテクチャ設計書

本ドキュメントでは、ライフプラン・シミュレーターの内部構造、データフロー、およびシミュレーションエンジンの仕組みについて解説します。

## 1. 状態管理 (State Management)

本アプリケーションは、**Zustand** を用いてグローバルな状態管理を行っています。データ構造は大きく2つに分かれます。

1.  **UserBaseParams (共通パラメータ)**
    *   現在の年齢、配偶者の有無、収入、インフレ率などの基本情報。
    *   `commonEvents`: すべてのシミュレーションプランに共通して適用されるライフイベント（出産、教育費、車の購入など）。
2.  **PlanState (個別プラン)**
    *   新築戸建、賃貸など、ユーザーが作成した個別のシミュレーションプラン。
    *   `SimulationParams`: 物件価格、ローン条件、固定資産税などの詳細パラメータ。ここには個別のライフイベント（`events`）も含まれます。
    *   `YearlyResult[]`: シミュレーションエンジンによって計算された、毎年のキャッシュフローと資産残高の結果配列。

### データの永続化とハイドレーション
Zustand の `persist` ミドルウェアを使用し、状態をブラウザの `localStorage` に保存しています。
Next.js の SSR（サーバーサイドレンダリング）とのハイドレーション不一致を防ぐため、`_hasHydrated` フラグを使用し、データが復元された後にのみメインのUI（グラフなど）をレンダリングする設計としています。また、データ復元時には `recalculateResults()` を呼び出し、最新のロジックで再計算を行ってから画面に表示します。

## 2. シミュレーションエンジン (`src/lib/engine.ts`)

UIから独立した純粋な関数 `runSimulation(params: SimulationParams)` として実装されています。主な計算ロジックは以下の通りです。

### 処理フロー
1.  **初期費用の計算**
    *   頭金、仲介手数料（中古物件のみ）、修繕積立基金（新築マンションのみ）などの初期費用を算出し、初年度の資産からマイナスします。
2.  **年次ループ計算**
    *   現在の年齢から「死亡想定年齢（例：90歳）」まで、1年ごとにループを回して以下の数値を算出します。
    *   **収入**: 給与上昇率を掛けた手取り収入を加算。（配偶者の収入も考慮）。リタイア年齢以降は収入を0（または年金等）として計算。
    *   **基本生活費**: インフレ率を掛けた生活費を減算。
    *   **住居費**:
        *   賃貸の場合：家賃 ＋ 更新料（按分計算）。
        *   購入の場合：ローン返済額（元利均等返済の計算式を使用） ＋ 維持費（固定資産税、管理費、修繕積立金、保険料）。
        *   ※修繕積立金は、設定された「5年ごとの上昇率（`repairReserveStepUpRate`）」に基づいて段階的に増加します。
    *   **住宅ローン控除**:
        *   借入残高の0.7%を計算し、物件種別（ZEH, 長期優良住宅など）と世帯属性（子育て世帯など）から判定した「控除上限額」および「期間（10年または13年）」に基づいて還付金を加算します。
    *   **ライフイベント**:
        *   共通イベントと個別イベントをマージし、該当する年の支出を減算します。
    *   **資産運用**:
        *   残った資産残高に対して、設定された「運用利回り」を掛けて運用益を加算します。
3.  **資産価値の評価 (出口戦略)**
    *   持ち家の場合、築年数と「建物の耐用年数（`buildingDepreciationYears`）」から建物の残存価値を直線法で減価償却し、土地価値と合わせた「不動産価値」を算出します。
    *   各年度の「総資産（金融資産 ＋ 不動産価値 － ローン残債）」を算出し、グラフに描画するための純資産としています。

## 3. コンポーネント構成

*   **`Sidebar.tsx`**: 左側の操作パネル。インフレ率や運用利回りなど、全プランに影響を与える共通パラメータを操作します。Phase 9のアップデートにより「共通ライフプラン（子育てセット・車購入）」のプリセット追加機能がここに集約されました。
*   **`page.tsx` (Dashboard)**: メイン画面。作成された複数プランのサマリーカード、純資産推移の折れ線グラフ（`SimChart`）、比較テーブルを表示します。
*   **`PlanDetailEditor.tsx`**: カードの歯車アイコンをクリックした際に開くモーダル。物件価格、ローン金利、2025年ローン控除の適用条件、修繕積立金の設定など、プラン個別の非常に細かい変数をチューニングするためのエディタです。個別のライフイベントもここで設定します。

## 4. 今後の拡張性について

*   **モンテカルロ・シミュレーション**: 現在は固定の利回り・インフレ率で計算していますが、`engine.ts` を拡張し、株価のボラティリティを考慮した確率分布（乱数発生）によるシミュレーション結果（ワーストケース、ベストケース）を追加することが構造上容易です。
*   **年金・税金計算の精緻化**: 現在、手取り収入は概算値として入力させていますが、額面給与から社会保険料や所得税を動的に控除し、さらにねんきん定期便のデータを反映できる拡張スロットがパラメーター上に残されています。
