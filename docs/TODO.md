## エクスポート機能（AI向けCSV出力）の実装

シミュレーション結果をChatGPTやClaudeなどのAI（またはデータ分析ツール）に読み込ませて、詳細な分析やアドバイスを受けられるようにするためのCSVエクスポート機能を実装します。

### 1. AIが読み取りやすいCSVの要件

AI（LLM）やPandasのようなデータ解析ライブラリが最も正確に文脈を理解し、計算・集計しやすいデータ構造（Tidy Data: 整然データ）を採用します。

**設計方針:**
*   **ロングフォーマット（縦長データ）の採用**: 1行を「1つのプランの、ある1年（年齢）のデータ」として記録します。横にプランを並べる（ワイドフォーマット）よりも、AIがフィルタリングや比較処理を行いやすくなります。
*   **フラットな構造**: ヘッダーは1行のみとし、セル結合などは行いません。
*   **明確な単位**: 数値はすべて「万円」で統一し、カンマを含まない純粋な数値（例: `4500`）として出力します。
*   **英語/ローマ字キーの検討（オプション）**: AIのトークン消費を抑え、プログラミング的処理を安定させるため、ヘッダーを英語（`year`, `age`, `plan_name`, `total_assets` 等）にするか、直感的な日本語にするかを検討します（今回は日本語ヘッダーでもLLMは十分に理解可能なため、わかりやすさを優先して日本語ヘッダーを基本とします）。

### 2. 出力するCSVデータのカラム設計（案）

| 列名 (Header) | 説明 (Description) | 例 (Example) |
| :--- | :--- | :--- |
| `プラン名` | 比較対象のプラン名称 | 新築戸建 |
| `西暦` | シミュレーションの西暦 | 2025 |
| `年齢` | その年のユーザーの年齢 | 35 |
| `総資産_万円` | その年の終了時点の総資産（金融資産＋不動産価値ーローン残債） | 4500 |
| `金融資産残高_万円` | その年の終了時点の現金・投資信託などの合計 | 1200 |
| `不動産価値_万円` | その年の終了時点の建物の残存価値と土地価値の合計 | 3300 |
| `手取り収入_万円` | その年の世帯手取り収入（運用益除く） | 650 |
| `基本生活費_万円` | その年のインフレ加味後の生活費 | 200 |
| `住居費合計_万円` | ローン返済、家賃、維持費の合計 | 240 |
| `ローン残債_万円` | 年末時点の住宅ローン残高 | 3400 |
| `住宅ローン控除額_万円` | その年の住宅ローン控除による還付額 | 24 |
| `ライフイベント支出_万円` | その年の一時的なイベント支出合計 | 50 |
| `運用益_万円` | その年の金融資産から得られた運用益 | 36 |

※ 上記のカラムを持つデータが、「比較しているプラン数 × シミュレーション年数」行出力されます。
