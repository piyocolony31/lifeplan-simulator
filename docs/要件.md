# ライフプランシミュレーター 要件定義書（Web版）

## 1. プロジェクト概要
本プロジェクトでは、「賃貸」「新築戸建」「中古戸建」「新築マンション」「中古マンション」の5つの住宅プランを軸に、将来の収支・資産推移を詳細にシミュレーション・比較できるウェブアプリケーションを開発します。
公的統計データに基づいた精緻な計算ロジックを備え、ユーザーが自身のライフイベントや経済状況に合わせた最適な住居選択を行えるよう支援することを目的とします。

## 2. システム要件

### 2.1 機能要件
- **複数プラン比較機能**: 
    - 5種類の住宅パターン（賃貸、新築/中古戸建、新築/中古マンション）を同一グラフ上で比較。
    - 各プランごとに個別のローン条件、修繕費、管理費などを設定可能。
- **生涯収支シミュレーション**:
    - 年次・月次ベースでの手取り収入、支出、貯蓄推移の算出。
    - 税金（所得税・住民税）および社会保険料の自動推計。
- **ライフイベント設定**:
    - 結婚、出産、教育（公立/私立選択）、転職、退職、介護、住宅買替などのイベント登録。
    - 各イベントに伴う一時的な支出増や定期的な収支変動の反映。
- **高度なシミュレーション**:
    - インフレ率、給与成長率、投資利回りの設定。
    - モンテカルロ法を用いた、不確実性（運用利回り変動、寿命等）を考慮した目標達成率の算出。
- **ビジュアライゼーション**:
    - 資産推移、年間収支の推移を折れ線グラフ/棒グラフで表示。
    - プランごとの色分け、ツールチップによる詳細数値表示、ズーム機能。
- **データ管理**:
    - 入力条件のJSON形式での保存・読み込み。
    - 計算結果のCSV/PDFエクスポート。
    - ブラウザへの自動保存（ローカルストレージ/IndexedDB）。

### 2.2 非機能要件
- **プラットフォーム**: ウェブブラウザ（Chrome, Edge, Safari, Firefoxの最新版）。
- **レスポンシブデザイン**: PC、タブレット、スマートフォン各デバイスでの快適な操作。
- **パフォーマンス**: シミュレーション計算およびグラフ描画を1秒以内に完了。
- **セキュリティ**: 個人を特定する情報のサーバー保存を避け、クライアントサイドでの処理・保存を基本とする。
- **拡張性**: 将来的な税制改正や新機能（保険提案、資産運用アドバイス等）を統合しやすいコンポーネント設計。

### 3.1 パターン別諸費用・独自項目
各プランの初期費用や維持費に以下の項目を自動計算または入力項目として加味する。

- **共通（購入）**:
    - **登記費用・司法書士報酬**: 物件価格の約0.5%〜1%
    - **印紙代**: 契約金額に応じた実費（数万円）
    - **融資手数料**: 借入額の2.2%（定率型）または一律（定額型）を選択
    - **火災・地震保険料**: 一括払いまたは年払いを選択
- **賃貸**:
    - **初期費用**: 敷金（1-2ヶ月）、礼金（1-2ヶ月）、仲介手数料（1ヶ月）、火災保険料、鍵交換費用
    - **更新料**: 2年ごとに家賃の1ヶ月分（※地域差あり、設定可能にする）
- **中古（戸建・マンション）**:
    - **仲介手数料**: (物件価格 × 3% + 6万円) × 消費税
- **新築マンション**:
    - **修繕積立基金**: 入居時に一括払い（数十万〜100万円程度）
- **戸建（新築・中古）**:
    - **修繕積立**: 自主積み立てとして年次支出に計上（マンションの精算金に相当）

## 4. 技術スタック（推奨）
本システムは「ウェブアプリケーション」として構築します。

- **Frontend**: React / Next.js (TypeScript)
- **Styling**: Tailwind CSS / CSS Modules
- **State Management**: Zustand / Recoil
- **Charts**: Chart.js / Recharts OR ECharts (高性能グラフ用)
- **Sim Engine**: クライアントサイドでのJavaScript実行（Web Workerを活用して描画を妨げない設計）
- **Storage**: IndexedDB (Local-first) / JSONファイル入出力

## 5. 計算ロジック

### 5.1 基本シミュレーション（年次更新）
```javascript
for (let year = startYear; year <= endYear; year++) {
    // 収入計算（給与成長率・配偶者合算）
    income = personalIncome * Math.pow(1 + salaryGrowth, year - startYear);
    if (hasSpouse) income += spouseIncome * Math.pow(1 + spouseSalaryGrowth, year - startYear);

    // 可処分所得（簡易的な所得税・住民税・社会保険料計算）
    tax = calculateTax(income);
    socialInsurance = calculateSocialInsurance(income);
    disposableIncome = income - (tax + socialInsurance);

    // 支出計算（インフレ率考慮）
    livingExpenses = currentLivingExpenses * Math.pow(1 + inflationRate, year - startYear);

    // 住宅費計算
    if (plan === 'RENT') {
        housingCost = rent * 12;
    } else {
        mortgagePayment = calculateMortgage(loanAmount, rate, period);
        maintenanceCost = propertyPrice * maintenanceRate;
        housingCost = mortgagePayment + maintenanceCost + propertyTax + managementFee;
    }

    // 教育費
    educationCost = calculateEducationCost(children);

    totalExpenses = livingExpenses + housingCost + educationCost + otherCosts;

    // 資産残高の更新（投資利回り考慮）
    investmentReturn = assetBalance * expectedReturnRate;
    assetBalance += (disposableIncome - totalExpenses) + investmentReturn;
}
```

### 5.2 住宅ローン返済公式（元利均等）
- **毎月の返済額**: `A * [r(1+r)^N] / [(1+r)^N - 1]`
    - `A`: 借入額
    - `r`: 月利（年利 / 12）
    - `N`: 返済回数（年数 * 12）

### 5.3 住宅ローン控除（2025年入居ルール対応）
- **控除率**: 0.7%（年末ローン残高に対して適用）
- **控除期間**:
    - 新築・買取再販: 13年間
    - 中古住宅: 10年間
- **借入限度額（2025年入居）**:
    - **新築（子育て・若者世帯）**: 長期優良5,000万 / ZEH 4,500万 / 省エネ 4,000万
    - **新築（一般世帯）**: 長期優良4,500万 / ZEH 3,500万 / 省エネ 3,000万
    - **中古**: 長期優良・ZEH・省エネ 3,000万 / 一般 2,000万
- **計算条件**: 各年次の「手取り収入（所得税・住民税控除後）」に対して、収めた合計税額を上限としてキャッシュバック。

### 5.3 確率シミュレーション（モンテカルロ）
- **手法**: 変数（期待利回り、物価、寿命等）に対して正規分布または対数正規分布を仮定し、1,000回以上の試行を行い資産残高の分布を算出します。
- **KPI**: 「◯歳時点での資産残高がプラスである確率（目標達成率）」を評価指標とします。

## 6. UI/UX デザイン方針
- **ダッシュボード形式**: 左側に操作パネル（パラメータ入力）、右側にメイングラフを配置。
- **インタラクティブ性**: スライダー操作により、リアルタイムでグラフが変動する直感的なUI。
- **比較の容易性**: プランのON/OFFを切り替えて、特定の組み合わせのみを視覚的に比較可能。
- **レスポンシブ対応**: モバイル（iOS/Android）およびデスクトップブラウザの双方で、グラフや表が最適化された表示。

## 7. 追加機能（オプション）
- **AIアドバイザー**: シミュレーション結果に基づき、「購入時期の推奨」や「予算上限の目安」をAIが提示。
- **税制改正アップデート**: 毎年、最新の税制や社会保険料率をサーバーから取得または反映。

## 8. 開発ロードマップ
1. **Phase 1**: 基本エンジン開発（単独プランの計算・表示）
2. **Phase 2**: マルチプラン比較グラフの実装

4. **Phase 4**: 確率シミュレーション・エクスポート機能・UI完成
